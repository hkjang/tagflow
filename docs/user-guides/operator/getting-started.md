# 운영자 시작하기

TagFlow RFID 태그 관리 시스템의 운영자 계정으로 시작하는 방법을 안내합니다.

## 운영자 역할 소개

운영자(Operator)는 일상적인 RFID 태그 모니터링과 리포트 조회를 담당하는 역할입니다.

### 운영자 권한

#### ✅ 가능한 작업

- 태그 이벤트 조회 및 모니터링
- 리포트 및 통계 조회
- 웹훅 로그 조회 (읽기 전용)
- 본인 비밀번호 변경

#### ❌ 불가능한 작업

- 사용자 관리
- 웹훅 생성/편집/삭제
- 웹훅 테스트
- 시스템 설정 변경
- 데이터 정리 작업

```
💡 참고
더 많은 권한이 필요한 경우 관리자에게 문의하세요.
```

## 첫 로그인

### 1. 계정 정보 받기

관리자로부터 다음 정보를 받습니다:

- 사용자명 (username)
- 임시 비밀번호

### 2. 로그인

1. TagFlow 애플리케이션 실행
2. 로그인 화면에서 사용자명과 비밀번호 입력
3. **"로그인"** 버튼 클릭

### 3. 비밀번호 변경 (권장)

첫 로그인 후 비밀번호를 변경하는 것이 좋습니다:

1. 우측 상단 사용자 메뉴 클릭
2. **"비밀번호 변경"** 선택
3. 현재 비밀번호 입력
4. 새 비밀번호 입력 및 확인
5. **"변경"** 버튼 클릭

**강력한 비밀번호 만들기**:

- 최소 8자 이상
- 영문 대소문자, 숫자, 특수문자 혼합
- 추측하기 어려운 조합

## 대시보드 둘러보기

로그인 성공 시 대시보드 화면이 표시됩니다.

### 화면 구성

#### 1. 상단 네비게이션 바

- **사용자 정보**: 현재 로그인한 사용자명과 역할 표시
- **로그아웃 버튼**: 시스템 종료
- **현재 시간**: 실시간 시계

#### 2. 사이드바 메뉴

운영자가 접근할 수 있는 메뉴:

```
📊 대시보드
  └─ 전체 현황 요약

🏷️ 태그 이벤트
  └─ RFID 스캔 이벤트 목록

📈 리포트
  ├─ 이벤트 통계
  └─ 웹훅 성능

🔗 웹훅 로그 (읽기 전용)
  └─ 웹훅 실행 기록
```

```
💡 참고
"관리" 메뉴는 관리자 전용이므로 운영자에게는 표시되지 않습니다.
```

#### 3. 메인 콘텐츠 영역

선택한 메뉴의 내용이 표시되는 중앙 화면입니다.

### 대시보드 위젯

운영자 대시보드에 표시되는 정보:

#### 오늘의 이벤트

```
오늘 처리된 RFID 태그 스캔 이벤트 총 개수
실시간으로 업데이트됩니다
```

#### 활성 웹훅

```
현재 활성화된 웹훅 개수
(설정 변경은 관리자만 가능)
```

#### 최근 이벤트

```
가장 최근에 수신된 태그 이벤트 목록 (최대 10개)
- 태그 UID
- 스캔 시간
- 장치 ID
- 위치
```

#### 웹훅 성공률

```
최근 웹훅 전송 성공률 그래프
성능 모니터링용
```

## 주요 메뉴 소개

### 태그 이벤트 페이지

**경로**: `/events`

RFID 태그 스캔 이벤트를 실시간으로 모니터링할 수 있습니다.

**주요 기능**:

- 이벤트 목록 조회 (시간 역순)
- 검색 및 필터링
- 실시간 업데이트
- 상세 정보 보기

**사용 방법**:

1. 사이드바에서 **"태그 이벤트"** 클릭
2. 이벤트 목록 확인
3. 검색창에서 특정 태그 UID 검색 가능
4. 날짜 필터로 특정 기간 조회

자세한 내용은 [태그 이벤트 모니터링](tag-event-monitoring.md)을 참조하세요.

### 리포트 페이지

**경로**: `/reports`

통계 데이터와 성능 지표를 확인할 수 있습니다.

**이벤트 통계**:

- 시간대별 이벤트 수
- 장치별 이벤트 분포
- 위치별 이벤트 분포
- 일일/주간/월간 트렌드

**웹훅 성능**:

- 웹훅별 성공률
- 평균 응답 시간
- 실패 비율
- 재시도 통계

자세한 내용은 [리포트 보기](viewing-reports.md)를 참조하세요.

### 웹훅 로그 페이지

**경로**: `/webhooks/logs` (읽기 전용)

웹훅 실행 기록을 조회할 수 있습니다.

```
⚠️ 읽기 전용
운영자는 웹훅 로그를 조회만 할 수 있습니다.
웹훅 설정 변경은 관리자만 가능합니다.
```

**조회 가능 정보**:

- 웹훅 전송 시간
- 대상 URL
- HTTP 상태 코드
- 응답 시간
- 재시도 횟수
- 오류 메시지 (실패 시)

자세한 내용은 [웹훅 로그](webhook-logs.md)를 참조하세요.

## 일상 업무 워크플로우

### 하루 시작 시

1. **로그인**

   ```
   애플리케이션 실행 → 로그인
   ```

2. **대시보드 확인**

   ```
   오늘의 이벤트 수, 시스템 상태 한눈에 파악
   ```

3. **최근 이벤트 검토**
   ```
   대시보드의 "최근 이벤트" 위젯으로 빠른 점검
   ```

### 모니터링 중

1. **태그 이벤트 실시간 모니터링**

   ```
   태그 이벤트 페이지에서 실시간 업데이트 확인
   ```

2. **이상 징후 확인**

   ```
   - 특정 시간대 이벤트 급증/급감
   - 특정 장치의 이벤트 중단
   - 웹훅 실패율 증가
   ```

3. **필요 시 검색 및 필터링**
   ```
   특정 태그 UID, 날짜, 장치로 필터링하여 상세 조회
   ```

### 정기 보고

1. **리포트 페이지 접속**

   ```
   주간/월간 리포트 생성
   ```

2. **통계 데이터 수집**

   ```
   - 기간별 이벤트 수
   - 장치별 활용도
   - 웹훅 성능 지표
   ```

3. **이상 패턴 식별**
   ```
   평소와 다른 패턴 발견 시 관리자에게 보고
   ```

## 자주 사용하는 기능

### 태그 UID 검색

특정 RFID 태그의 이력을 조회하려면:

1. **태그 이벤트** 페이지로 이동
2. 검색창에 태그 UID 입력
3. 해당 태그의 모든 스캔 기록 확인

### 날짜 범위 필터링

특정 기간의 데이터를 보려면:

1. 날짜 필터 클릭
2. 시작 날짜와 종료 날짜 선택
3. **"적용"** 버튼 클릭
4. 해당 기간의 데이터만 표시됨

### 이벤트 상세 정보

특정 이벤트의 전체 정보를 보려면:

1. 이벤트 목록에서 원하는 이벤트 클릭
2. 상세 정보 팝업 또는 패널 표시
3. 모든 필드 값 확인 가능

## 비밀번호 관리

### 비밀번호 변경

정기적으로 비밀번호를 변경하세요 (3개월마다 권장):

1. 우측 상단 **사용자 메뉴** 클릭
2. **"비밀번호 변경"** 선택
3. 현재 비밀번호 입력
4. 새 비밀번호 입력 및 확인
5. **"변경"** 버튼 클릭

### 비밀번호를 잊어버렸을 때

운영자는 본인의 비밀번호를 재설정할 수 없습니다.

**해결 방법**:

1. 관리자에게 연락
2. 관리자가 임시 비밀번호 설정
3. 임시 비밀번호로 로그인
4. 즉시 새 비밀번호로 변경

## 문제 해결

### 로그인할 수 없어요

**증상**: 사용자명과 비밀번호를 입력해도 로그인되지 않습니다.

**해결 시도**:

1. 사용자명과 비밀번호 정확히 입력했는지 확인
2. 대소문자 구분 주의
3. Caps Lock 키 확인
4. 임시 비밀번호인 경우 관리자가 준 그대로 입력
5. 문제 지속 시 관리자에게 문의

### 메뉴가 보이지 않아요

**증상**: 가이드에 설명된 메뉴가 보이지 않습니다.

**확인 사항**:

1. "관리" 메뉴는 운영자에게 표시되지 않음 (정상)
2. 페이지 새로고침 시도 (F5)
3. 로그아웃 후 재로그인
4. 문제 지속 시 관리자에게 문의

### 데이터가 업데이트되지 않아요

**증상**: 새로운 태그 이벤트가 목록에 나타나지 않습니다.

**해결 시도**:

1. 페이지 새로고침 (F5)
2. 백엔드 서버 상태 확인 (관리자에게 문의)
3. 네트워크 연결 확인
4. 브라우저 캐시 삭제 후 재시도

### 검색이 작동하지 않아요

**증상**: 검색창에 입력해도 결과가 없습니다.

**확인 사항**:

1. 정확한 태그 UID 입력했는지 확인
2. 날짜 필터가 적용되어 있는지 확인
3. 대소문자 구분 (시스템에 따라 다름)
4. 와일드카드 검색 지원 여부 확인

## 도움말 및 지원

### 추가 정보

운영자로서 알아두면 유용한 문서:

1. **[태그 이벤트 모니터링](tag-event-monitoring.md)**

   - 이벤트 조회, 검색, 필터링 상세 가이드

2. **[리포트 보기](viewing-reports.md)**

   - 통계 리포트 활용 방법

3. **[웹훅 로그](webhook-logs.md)**
   - 웹훅 실행 기록 해석 방법

### 관리자에게 문의할 사항

다음과 같은 경우 관리자에게 도움을 요청하세요:

- 비밀번호를 잊어버린 경우
- 더 많은 권한이 필요한 경우
- 시스템 오류 또는 버그 발견
- 웹훅 설정 변경 요청
- 새로운 기능 제안

## 보안 주의사항

### 계정 보안

- 비밀번호를 다른 사람과 공유하지 마세요
- 공용 컴퓨터에서 사용 후 반드시 로그아웃
- 이석 시 화면 잠금
- 의심스러운 활동 발견 시 즉시 관리자에게 보고

### 데이터 보안

- 민감한 태그 데이터를 외부로 유출하지 마세요
- 리포트 데이터를 외부로 전송할 때는 승인 받기
- 화면 캡처 시 민감 정보 마스킹

## 효율적인 운영 팁

### 단축키 활용

- **F5**: 페이지 새로고침
- **Ctrl + F**: 페이지 내 검색
- **Esc**: 팝업 닫기

### 북마크 활용

자주 사용하는 페이지를 브라우저에 북마크:

- 태그 이벤트 페이지
- 리포트 페이지
- 대시보드

### 정기 점검 루틴

**매일**:

- 대시보드 확인
- 최근 이벤트 검토

**매주**:

- 주간 리포트 생성 및 검토
- 이상 패턴 식별

**매월**:

- 월간 통계 리포트
- 트렌드 분석
