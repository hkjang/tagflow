# 태그 이벤트 모니터링

RFID 태그 스캔 이벤트를 실시간으로 조회하고 모니터링하는 방법을 안내합니다.

## 태그 이벤트 페이지 접근

1. 사이드바에서 **"태그 이벤트"** (또는 **"Events"**) 클릭
2. 모든 RFID 태그 스캔 이벤트 목록이 표시됩니다

## 이벤트 목록 이해하기

### 표시되는 정보

각 이벤트는 다음 정보를 포함합니다:

| 필드           | 설명                    | 예시                |
| -------------- | ----------------------- | ------------------- |
| **ID**         | 이벤트 고유 번호        | 1234                |
| **태그 UID**   | RFID 태그 고유 식별자   | TAG_AB123456        |
| **타임스탬프** | 스캔된 시각             | 2025-12-03 14:30:25 |
| **장치 ID**    | RFID 리더 장치 식별자   | READER_01           |
| **위치**       | 스캔 위치 (설정된 경우) | 창고 A동 1층        |

### 정렬

기본적으로 이벤트는 **최신순**(시간 역순)으로 정렬됩니다.

- 가장 최근 스캔된 이벤트가 맨 위에 표시
- 실시간으로 새 이벤트가 추가됨

## 검색 및 필터링

### 태그 UID로 검색

특정 RFID 태그의 이력을 조회하려면:

1. 검색창에 태그 UID 입력

   ```
   예: TAG_AB123456
   ```

2. Enter 키 또는 검색 버튼 클릭

3. 해당 태그의 모든 스캔 기록만 표시됨

### 날짜 범위 필터

특정 기간의 이벤트만 보려면:

1. **날짜 필터** 버튼 클릭

2. **시작 날짜** 선택

   ```
   예: 2025-12-01
   ```

3. **종료 날짜** 선택

   ```
   예: 2025-12-03
   ```

4. **"적용"** 버튼 클릭

5. 선택한 기간의 이벤트만 표시됩니다

### 장치별 필터

특정 RFID 리더의 이벤트만 보려면:

1. **장치 필터** 드롭다운 클릭

2. 원하는 장치 선택

   ```
   예: READER_01
   ```

3. 해당 장치의 이벤트만 표시됨

### 위치별 필터

특정 위치의 이벤트만 보려면:

1. **위치 필터** 드롭다운 클릭

2. 원하는 위치 선택

   ```
   예: 창고 A동 1층
   ```

3. 해당 위치의 이벤트만 표시됨

### 복합 필터

여러 필터를 동시에 적용할 수 있습니다:

```
예시: 2025-12-01부터 2025-12-03까지, READER_01에서 스캔된,
창고 A동 1층의 모든 이벤트
```

## 실시간 모니터링

### 자동 업데이트

태그 이벤트 페이지는 자동으로 새 이벤트를 표시합니다:

- 새로운 태그 스캔 발생 시 자동으로 목록에 추가
- 수동 새로고침 불필요
- **실시간** 아이콘이 활성화되어 있는지 확인

### 새로고침

필요시 수동으로 새로고침:

1. **새로고침** 버튼 클릭
2. 또는 **F5** 키 누르기

## 이벤트 상세 정보

### 상세 보기

특정 이벤트의 전체 정보를 보려면:

1. 이벤트 행 클릭
2. 상세 정보 팝업 또는 패널 표시

### 상세 정보 내용

**기본 정보**:

- 이벤트 ID
- 태그 UID
- 타임스탬프 (정확한 시간, 초 단위)
- 장치 ID
- 위치

**추가 메타데이터** (있는 경우):

- 리더 모델
- 신호 강도 (RSSI)
- 안테나 번호
- 사용자 정의 필드

**웹훅 전송 정보**:

- 해당 이벤트로 트리거된 웹훅 목록
- 웹훅 전송 성공/실패 상태
- 웹훅 로그 링크

## 이벤트 이해하기

### 정상 이벤트

일반적인 RFID 태그 스캔:

```
ID: 1234
태그 UID: TAG_AB123456
타임스탬프: 2025-12-03 14:30:25
장치: READER_01
위치: 창고 A동
```

### 중복 이벤트

같은 태그가 짧은 시간 내에 여러 번 스캔된 경우:

```
ID: 1235
태그 UID: TAG_AB123456    ← 위와 동일한 태그
타임스탬프: 2025-12-03 14:30:27  ← 2초 차이
장치: READER_01
위치: 창고 A동
```

이는 정상 동작일 수 있습니다:

- RFID 리더가 연속으로 스캔
- 태그가 읽기 범위 내에 계속 있음

### 비정상 패턴

주의가 필요한 패턴:

**1. 이벤트 급증**

```
짧은 시간에 수백/수천 개의 이벤트
→ 리더 오작동 또는 설정 문제 가능성
```

**2. 이벤트 중단**

```
특정 장치에서 오랜 시간 이벤트 없음
→ 장치 오프라인 또는 네트워크 문제
```

**3. 알 수 없는 태그**

```
등록되지 않은 태그 UID 스캔
→ 새 태그 또는 외부 태그 감지
```

이러한 패턴 발견 시 관리자에게 보고하세요.

## 페이지네이션

### 페이지 탐색

이벤트가 많을 경우 여러 페이지로 나뉩니다:

- **다음 페이지**: 오른쪽 화살표 또는 "다음" 버튼
- **이전 페이지**: 왼쪽 화살표 또는 "이전" 버튼
- **특정 페이지**: 페이지 번호 직접 클릭

### 페이지당 항목 수

한 페이지에 표시할 이벤트 수를 조정할 수 있습니다:

```
옵션: 10, 25, 50, 100개
기본값: 25개
```

많은 이벤트를 한번에 보려면 100개로 설정하세요.

## 데이터 내보내기

현재 버전에서는 UI를 통한 데이터 내보내기를 지원하지 않습니다.

필요시 관리자에게 요청하세요:

- CSV 파일 내보내기
- Excel 파일 내보내기
- API를 통한 데이터 추출

## 사용 시나리오

### 시나리오 1: 특정 물품 추적

**목표**: 태그 `TAG_ITEM001`의 이동 경로 확인

**단계**:

1. 검색창에 `TAG_ITEM001` 입력
2. 모든 스캔 기록 확인
3. 각 이벤트의 위치 정보로 이동 경로 파악

**예상 결과**:

```
14:00 - 입고장
14:15 - 창고 A동 1층
14:30 - 분류장
15:00 - 출하장
```

### 시나리오 2: 특정 시간대 활동 모니터링

**목표**: 오전 10시~12시 사이 모든 활동 확인

**단계**:

1. 날짜 필터 → 오늘 날짜 선택
2. 시간 필터 → 10:00 ~ 12:00 (지원되는 경우) 3.또는 전체 목록에서 시간 직접 확인

**활용**:

- 특정 시간대 작업량 확인
- 피크 시간 식별
- 업무 패턴 분석

### 시나리오 3: 장치 상태 점검

**목표**: `READER_02` 장치가 정상 작동하는지 확인

**단계**:

1. 장치 필터 → `READER_02` 선택
2. 최근 이벤트 확인

**판단 기준**:

- 최근 1시간 내 이벤트 있음 → 정상
- 4시간 이상 이벤트 없음 → 점검 필요

### 시나리오 4: 이상 활동 탐지

**목표**: 정규 근무 시간 외 비정상 활동 감지

**단계**:

1. 날짜 필터 → 어제 또는 주말 날짜
2. 이벤트 목록 검토
3. 예상치 못한 이벤트 확인

**주의사항**:

- 야간 근무가 있는 경우 정상일 수 있음
- 관리자에게 확인 후 판단

## 문제 해결

### 이벤트가 표시되지 않아요

**증상**: 이벤트 목록이 비어있습니다.

**확인 사항**:

1. 실제로 태그 스캔이 발생했는지 확인
2. 필터가 과도하게 적용되어 있는지 확인
   - 날짜 필터 해제
   - 검색어 초기화
3. 페이지 새로고침 (F5)
4. 백엔드 서버 상태 (관리자에게 문의)

### 실시간 업데이트가 안 돼요

**증상**: 새 이벤트가 자동으로 추가되지 않습니다.

**해결 시도**:

1. "실시간" 토글이 활성화되어 있는지 확인
2. 페이지 새로고침
3. 네트워크 연결 확인
4. 브라우저 콘솔 오류 확인 (F12)

### 검색 결과가 없어요

**증상**: 태그 UID로 검색했는데 결과가 없습니다.

**확인 사항**:

1. 태그 UID를 정확히 입력했는지 확인
2. 대소문자 구분 (시스템에 따라)
3. 해당 태그가 실제로 스캔된 적이 있는지 확인
4. 날짜 필터 범위 확인

### 페이지가 느려요

**증상**: 이벤트 목록 로딩이 느립니다.

**원인과 해결**:

**원인 1**: 대량의 이벤트

```
해결: 날짜 필터로 범위 축소
```

**원인 2**: 네트워크 지연

```
해결: 네트워크 상태 확인
```

**원인 3**: 브라우저 성능

```
해결:
- 브라우저 캐시 삭제
- 다른 탭 닫기
- 페이지당 항목 수 줄이기 (100 → 25)
```

## 모범 사례

### 효율적인 모니터링

**1. 필터 활용**

```
불필요한 데이터는 필터로 제외하여 집중
```

**2. 정기 점검**

```
매일 특정 시간에 전일 이벤트 검토
```

**3. 북마크 활용**

```
자주 사용하는 필터 조합을 URL로 북마크
```

### 데이터 해석

**1. 컨텍스트 고려**

```
이벤트를 단독으로 보지 말고 전후 맥락 파악
```

**2. 패턴 인식**

```
정상 패턴을 숙지하여 이상 징후 빠르게 감지
```

**3. 교차 검증**

```
의심스러운 이벤트는 웹훅 로그와 교차 확인
```
