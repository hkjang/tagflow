# 리포트 보기

TagFlow의 통계 리포트와 성능 지표를 조회하고 분석하는 방법을 안내합니다.

## 리포트 페이지 접근

1. 사이드바에서 **"리포트"** 클릭
2. 이벤트 통계와 웹훅 성능 지표가 표시됩니다

## 이벤트 통계

### 개요 대시보드

**오늘의 통계**:

- 오늘 처리된 총 이벤트 수
- 시간당 평균 이벤트 수
- 가장 많이 스캔된 태그 TOP 5
- 가장 활성화된 리더 장치 TOP 5

### 시간대별 이벤트 차트

**시간별 분포** (24시간):

```
막대 그래프로 표시
X축: 시간 (0시~23시)
Y축: 이벤트 수
```

**활용**:

- 피크 시간대 식별
- 업무 패턴 분석
- 이상 시간대 감지

### 일별 트렌드

**최근 7일/30일 이벤트** 추세:

```
꺾은선 그래프로 표시
X축: 날짜
Y축: 일일 이벤트 수
```

**활용**:

- 일일 변동 확인
- 주간/월간 패턴 파악
- 증가/감소 추세 분석

### 장치별 통계

**RFID 리더별 이벤트 분포**:

```
파이 차트 또는 막대 그래프
각 장치의 이벤트 수와 비율
```

**정보**:
| 장치 ID | 이벤트 수 | 비율 |
|---------|----------|------|
| READER_01 | 1,234 | 45% |
| READER_02 | 890 | 32% |
| READER_03 | 632 | 23% |

**활용**:

- 장치 활용도 파악
- 부하 분산 검토
- 장치 성능 비교

### 위치별 통계

**위치별 이벤트 분포**:

```
각 위치에서 발생한 이벤트 수
```

**정보**:
| 위치 | 이벤트 수 | 비율 |
|------|----------|------|
| 창고 A동 | 1,450 | 52% |
| 창고 B동 | 856 | 31% |
| 입출고장 | 470 | 17% |

**활용**:

- 위치별 활동량 파악
- 공간 활용도 분석
- 동선 최적화 검토

## 웹훅 성능 지표

### 웹훅 개요

**전체 통계**:

- 활성 웹훅 수
- 총 전송 횟수
- 전체 성공률
- 평균 응답 시간

### 웹훅별 성능

**각 웹훅의 상세 지표**:

| 웹훅 이름  | 전송 수 | 성공률 | 평균 응답 시간 | 실패 수 |
| ---------- | ------- | ------ | -------------- | ------- |
| ERP 연동   | 1,234   | 98.5%  | 145ms          | 19      |
| Slack 알림 | 1,234   | 99.8%  | 89ms           | 2       |
| 백업 서버  | 1,234   | 95.2%  | 312ms          | 59      |

**색상 코딩**:

- 🟢 녹색: 성공률 95% 이상
- 🟡 노랑: 성공률 90~95%
- 🔴 빨강: 성공률 90% 미만

### 응답 시간 분석

**웹훅 응답 시간 차트**:

```
꺾은선 그래프
X축: 시간
Y축: 응답 시간 (ms)
```

**임계값**:

- 우수: <100ms
- 양호: 100~300ms
- 주의: 300~1000ms
- 경고: >1000ms

### 실패 분석

**실패 원인 분류**:

```
파이 차트
- 타임아웃: 45%
- HTTP 5xx 오류: 30%
- HTTP 4xx 오류: 15%
- 네트워크 오류: 10%
```

**최근 실패 목록**:
| 시간 | 웹훅 | 오류 코드 | 오류 메시지 |
|------|------|----------|------------|
| 14:30 | ERP 연동 | 504 | Gateway Timeout |
| 14:25 | 백업 서버 | 500 | Internal Server Error |

## 맞춤 리포트

### 기간 선택

**리포트 기간 설정**:

1. 리포트 상단의 날짜 선택기 클릭
2. 시작 날짜와 종료 날짜 선택
3. **"적용"** 버튼 클릭
4. 선택한 기간의 리포트 생성

**사전 정의된 기간**:

- 오늘
- 어제
- 최근 7일
- 최근 30일
- 이번 달
- 지난 달

### 필터링

**특정 데이터만 분석**:

- 특정 장치만
- 특정 위치만
- 특정 웹훅만

## 리포트 활용 사례

### 일일 운영 보고

**목표**: 매일 업무 종료 시 일일 리포트 작성

**단계**:

1. 리포트 페이지 접속
2. 기간을 "오늘"로 설정
3. 주요 지표 확인:
   ```
   - 총 이벤트 수
   - 피크 시간대
   - 장치별 분포
   - 웹훅 성공률
   ```
4. 이상 항목 식별 및 기록

**보고서 작성**:

```
일일 운영 보고서 (2025-12-03)
----------------------------
총 이벤트: 2,756건
피크 시간: 14:00~15:00 (342건)
주요 장치: READER_01 (45%)
웹훅 성공률: 97.8%
이상사항: READER_03 응답시간 증가 (평균 450ms)
```

### 주간 트렌드 분석

**목표**: 주간 단위로 트렌드 파악

**단계**:

1. 기간을 "최근 7일"로 설정
2. 일별 트렌드 차트 검토
3. 전주 대비 증감 확인

**분석 포인트**:

```
- 요일별 패턴 (평일 vs 주말)
- 증가/감소 추세
- 이상 급증/급감일 확인
```

### 월간 성능 리포트

**목표**: 월말에 전체 성능 평가

**단계**:

1. 기간을 "이번 달"로 설정
2. 모든 지표 종합 검토
3. 주요 KPI 정리

**주요 KPI**:

```
- 월간 총 이벤트 수
- 일평균 이벤트 수
- 웹훅 전체 성공률
- 평균 응답 시간
- 장치별 가동률
```

### 문제 해결 분석

**목표**: 웹훅 실패 원인 파악

**단계**:

1. 웹훅 성능 섹션으로 이동
2. 실패율이 높은 웹훅 식별
3. 실패 원인 차트 확인
4. 최근 실패 목록 검토

**대응**:

```
- 타임아웃이 많으면 → 대상 서버 성능 점검
- 4xx 오류 → 인증 또는 URL 확인
- 5xx 오류 → 외부 시스템 문제 보고
```

## 리포트 해석

### 정상 패턴

**일반적인 정상 지표**:

**이벤트**:

- 일정한 패턴 유지
- 예측 가능한 피크 시간
- 큰 변동 없음

**웹훅**:

- 성공률 95% 이상
- 응답 시간 300ms 이하
- 실패 재시도로 대부분 해결

### 주의가 필요한 패턴

**이벤트 이상**:

```
⚠️ 급격한 증가/감소 (50% 이상)
⚠️ 특정 장치 이벤트 중단
⚠️ 예상치 못한 시간대 활동
```

**웹훅 이상**:

```
⚠️ 성공률 90% 미만
⚠️ 응답 시간 1초 이상
⚠️ 특정 웹훅 지속적 실패
```

### 조치 가이드

**이벤트 급증**:

```
1. 장치별 분포 확인 → 특정 장치 문제인지 파악
2. 시간대 확인 → 일시적인지 지속적인지 판단
3. 관리자에게 보고
```

**웹훅 성능 저하**:

```
1. 특정 웹훅인지 전체인지 확인
2. 실패 원인 분석
3. 필요시 관리자에게 외부 시스템 점검 요청
```

## 데이터 내보내기

현재 버전에서는 리포트 내보내기를 지원하지 않습니다.

**대안**:

- 화면 캡처 (주요 차트)
- 수동으로 데이터 기록
- 관리자에게 CSV 내보내기 요청

## 문제 해결

### 리포트가 표시되지 않아요

**증상**: 리포트 페이지가 비어있거나 로딩 중입니다.

**해결 시도**:

1. 페이지 새로고침 (F5)
2. 날짜 범위 확인 (너무 오래된 기간은 데이터 없을 수 있음)
3. 백엔드 서버 상태 확인 (관리자에게 문의)

### 차트가 올바르지 않아요

**증상**: 차트 데이터가 이상하거나 비어있습니다.

**확인 사항**:

1. 해당 기간에 실제 데이터가 있는지 확인
2. 필터 설정 검토
3. 브라우저 캐시 삭제 후 재시도

### 성능 지표가 업데이트되지 않아요

**증상**: 리포트 데이터가 최신이 아닙니다.

**해결**:

1. 새로고침 버튼 클릭
2. 날짜 범위 재설정
3. 로그아웃 후 재로그인

## 모범 사례

### 정기 모니터링

**일일**:

- 오늘 리포트 확인
- 웹훅 성공률 점검

**주간**:

- 7일 트렌드 분석
- 이상 패턴 식별

**월간**:

- 월간 종합 리포트 작성
- KPI 달성 여부 평가

### 효과적인 분석

**1. 비교 분석**:

```
전일/전주/전월 대비 비교로 변화 감지
```

**2. 컨텍스트 고려**:

```
특정 날짜 (공휴일, 특별 이벤트 등) 고려
```

**3. 종합 판단**:

```
단일 지표가 아닌 여러 지표 종합 분석
```
